# Demo Sequence File

这是一个用于验证 XingCode skills 机制的测试 skill：它默认不读取代码，只按用户要求顺序输出内容，或者生成一个简单测试文件。

## 目标

用最小、最稳定的方式验证以下链路是否正常：

1. project skill discovery
2. prompt 中出现 skill 摘要
3. 大模型按需调用 `load_skill`
4. 根据 skill 内容执行简单任务
5. 可选地调用 `write_file` 生成测试文件

## 使用规则

1. 默认不要读取仓库代码，不要调用 `list_files`、`read_file`、`run_command`，除非用户明确要求。
2. 如果用户要求“按顺序输出”，直接输出编号列表即可。
3. 如果用户要求“生成测试文件”，优先使用 `write_file`。
4. 如果用户没有给测试文件路径，默认写入当前工作目录下的 `skill_demo_output.txt`。
5. 如果用户同时要求“按序输出”和“生成文件”，先生成文件，再在回答里说明文件路径和内容摘要。
6. 如果用户要求的内容非常简单，不要过度发挥，不要自行扩展任务。

## 工作流

### 模式一：按序输出

当用户要求：

- 按顺序输出
- 列一个 1/2/3 清单
- 给出测试步骤

你应该：

1. 识别用户想要几个步骤
2. 如果用户没说数量，默认输出 3 步
3. 用编号列表直接输出
4. 每一步保持简短清晰

### 模式二：生成测试文件

当用户要求：

- 生成测试文件
- 写一个 demo 文件
- 输出到 txt 文件

你应该：

1. 确定目标路径
2. 如果用户没给路径，使用 `skill_demo_output.txt`
3. 调用 `write_file`
4. 文件内容尽量简单，建议包含：
   - 标题
   - 用户请求摘要
   - 1/2/3 的测试步骤

## 输出模板建议

### 顺序输出模板

```text
1. 第一步
2. 第二步
3. 第三步
```

### 测试文件模板

```text
Demo Skill Output

Request:
<用户请求摘要>

Steps:
1. 第一步
2. 第二步
3. 第三步
```

## 示例

### 示例 1：只按序输出

用户说：

```text
请使用 demo-sequence-file skill，按顺序输出一个 3 步的测试流程
```

你应该直接输出 1/2/3 列表，不需要读代码。

### 示例 2：生成测试文件

用户说：

```text
请使用 demo-sequence-file skill，生成一个测试文件，内容是发布前检查清单
```

你应该：

1. 调用 `write_file`
2. 默认写到 `skill_demo_output.txt`
3. 然后告诉用户文件已生成

### 示例 3：指定路径生成文件

用户说：

```text
请使用 demo-sequence-file skill，把测试内容写入 outputs/checklist.txt
```

你应该使用用户指定路径，而不是默认路径。
